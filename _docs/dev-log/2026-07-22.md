`IMPLEMENTATION_GUIDE.md`를 확인해보니 두 컴포넌트는 0-7 디렉토리 생성 목록에만 이름이 있고, 실제 `.types.ts`/`.tsx` 코드 스펙이 없었다. `PLANNING.md`에도 목적/난이도(★☆☆) 정도만 있고 props 계약은 없는 상태.

- `PLANNING.md` 7-6: `MiniStatCell` = "라벨+값 초소형 블록, 순수 CSS", `RankedList` = "Top-N 순위 리스트, Badge 선택"

### 2. `dl-ops-dashboard` 스킬 문서로 실사용처 조사

`dl-ops-dashboard` 스킬의 `ch1_common_components.md`, `ch2~ch5_*.md`를 뒤져 두 컴포넌트가 실제로 어느 화면·어떤 데이터에 쓰이는지 정리했다.

- `MiniStatCell`: Ch3 Kafka(Leader Election/Rebalance Events/Replication Health%/Controller Health, Segment Storage, Log & Retention), Ch4 Spark(Cluster stats block, Trino Refresh Status, Small File Compaction/Cluster Overall Health), Ch5 PPS/MinIO(Replication State, Stale/Small File Tables, Active Readers/Writers) 등에서 "라벨 하나 + 값 하나"를 그리드로 나열하는 용도로 반복 사용됨을 확인.
- `RankedList`: Ch2 Home(Recent Top-5 Query Ranking), Ch3 Kafka(Consumer Groups list, Sinks/Downstream list), Ch5 PPS/MinIO(Top Writers) — "이름 + 값" 쌍을 순위 순서로 보여주는 용도.

### 3. `IMPLEMENTATION_GUIDE.md`에 Phase 5b 설계/구현 계획 추가

Phase 5(파이프라인/토폴로지) 완료 체크와 Phase 6(대시보드 조립) 사이에 **Phase 5b — 기타 시각화 컴포넌트**를 신설했다. "6번 전이 낫다"는 판단 — Phase 6에서 두 컴포넌트를 실제 화면에 꽂아 넣으려면 그 전에 구현이 끝나 있어야 하기 때문.

- **5b-0 설계 검토**: 조사한 실사용처를 표로 정리하고, 값이 "순수 숫자 / 단위 붙은 숫자 / 상태 텍스트" 3종류로 섞여 있다는 관찰에서 `status?: StatusLevel` 선택 prop 설계를 도출. 그리드/리스트 배치 자체는 컴포넌트가 아니라 소비하는 화면(`SectionPanel` 내부)이 책임진다는 원칙도 명시.
- **5b-1 `MiniStatCell`**: types/tsx/index/stories 전체 코드 작성 (Default/WithUnit/Critical/Loading/Grid 스토리)
- **5b-2 `RankedList`**: types/tsx/index/stories 전체 코드 작성 (TopWriters/ConsumerGroupsByLag/DownstreamStatus/Loading/Empty 스토리, `getLagStatus` 계열 데이터 반영)
- **Phase 5b 완료 체크**: Storybook에서 확인할 항목(그리드 배치, 상태별 색상, Skeleton) 정리

참고 파일: `_docs/IMPLEMENTATION_GUIDE.md` (Phase 5b 섹션)

### 4. `MiniStatCell` 구현 (직접 작성)

Phase 5b-1 설계를 그대로 따라 `src/components/misc/MiniStatCell/` 아래 4개 파일을 구현했다.

- `MiniStatCell.types.ts`: `label`, `value`(number|string), `unit?`, `status?: StatusLevel`, `isLoading?`
- `MiniStatCell.tsx`: `status`가 있을 때만 값 텍스트에 `STATUS_COLORS[status]`를 인라인 스타일로 입히고, 없으면 `text-foreground` 뉴트럴 톤 사용. `isLoading`일 때 라벨/값 자리에 `Skeleton` 2개 렌더링.
- `MiniStatCell.stories.tsx`: `Default`(순수 카운트), `WithUnit`(퍼센트+normal 상태), `Critical`(퍼센트+critical 상태), `Loading`, `Grid`(4개를 `grid-cols-4`로 나열 — Ch4 Spark Cluster stats block 같은 실사용 배치를 재현)
- `index.ts`: 컴포넌트/타입 re-export

가이드에 적어둔 설계와 실제 구현이 100% 일치했고, `MiniStatCell.tsx`의 import 순서(Skeleton → types → STATUS_COLORS)만 가이드 코드와 미세하게 다른 정도.

참고 파일: `src/components/misc/MiniStatCell/`
