## 🐛 발생한 문제 & 해결

### 문제 1: recharts v3에서 `ResponsiveContainer` height 타입 에러

- **원인**: `IMPLEMENTATION_GUIDE.md`는 recharts v2 기준으로 작성됐는데 실제 설치 버전은 3.9.2. `ResponsiveContainer`의 `height` prop 타입이 `number | \`${number}%\`` 로 더 엄격해져, `ChartBaseProps.height`(`number | string`)를 그대로 넘기면 타입 에러 발생.
- **해결**: 세 recharts 변형 파일 모두 `height={height as number}`로 캐스트. 실사용상 height는 항상 숫자로 전달되므로 안전한 캐스트.
- **참고**: `npx tsc -b --noEmit`으로 확인.

### 문제 2: 제네릭 컴포넌트(`StatusDataTable<T>`) 스토리 작성 패턴 부재

- **원인**: 가이드 문서에 `StatusDataTable` 전용 스토리 예시가 없었음(`AlertEventTable` 예시만 존재).
- **해결**: 프로젝트의 기존 스토리 패턴(Default/Loading/Empty + 이모지 그룹 타이틀 `📋 Tables/...`)을 그대로 따라 Kafka 브로커 목업 데이터로 직접 구성.
